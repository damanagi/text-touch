# Text Touch v0.5 — API Contract (단일 진실 원천)

> 5명 에이전트가 병렬 작업하는 동안 모듈 간 경계면을 미리 동결한다. 변경 시 team lead 승인 후 이 파일을 먼저 갱신하고 다른 에이전트에게 공유.

## 1. IPC 채널 — 새로 추가되는 것

| 채널 | 방향 | 페이로드 | 응답 | 담당 |
|------|-----|---------|------|------|
| `win:confirmClose` | R→M | `{dirty: boolean, fileName: string\|null}` | `{action: 'save'\|'discard'\|'cancel'}` | electron-architect |
| `file:listBackups` | R→M | `{originalPath: string}` | `[{path, mtime, size}]` | fs-keeper |
| `file:restoreBackup` | R→M | `{backupPath: string, targetPath: string}` | `{success, restored: true}` | fs-keeper |
| `file:revealInFinder` | R→M | `{path: string}` | (없음) | electron-architect (shell.showItemInFolder) |

기존 6채널은 그대로. `file:saveOriginal` 응답에서 `backupPath`는 이제 `~/Library/Application Support/Text Touch/backups/{hash}/{ts}.html` 형태.

## 3. index.html 새 요소 (editor-engineer가 추가)

```html
<!-- 상단 서브툴바 (toolbar 아래) -->
<div id="format-bar" hidden>
  <select id="fmt-font-family" data-format-cmd="fontFamily">...</select>
  <select id="fmt-font-size" data-format-cmd="fontSize">...</select>
  <button data-format-cmd="bold" aria-label="굵게">B</button>
  <button data-format-cmd="italic" aria-label="기울임">I</button>
  <button data-format-cmd="underline" aria-label="밑줄">U</button>
  <button data-format-cmd="alignToggle" aria-label="정렬">⫶</button>
  <input type="color" data-format-cmd="foreColor" aria-label="글자 색" />
</div>

<!-- 미니 툴바 (셸 body 끝, 부동) -->
<div id="mini-toolbar" hidden role="toolbar" aria-label="텍스트 서식">
  <!-- mini-toolbar.js가 채움 -->
</div>

<!-- 찾기·바꾸기 바 (셸 toolbar 아래, hidden) -->
<div id="find-bar" hidden role="search">
  <!-- find-replace.js가 채움 -->
</div>
```

editor-engineer는 위 컨테이너만 마크업으로 보장. 내부는 각 모듈이 채움.

`data-format-cmd` 값 명세:
- `bold`, `italic`, `underline`, `strikethrough`, `superscript`, `subscript`
- `fontFamily`, `fontSize`, `foreColor`, `hiliteColor`
- `alignLeft`, `alignCenter`, `alignRight`, `alignJustify`, `alignToggle`
- `insertUnorderedList`, `insertOrderedList`, `indent`, `outdent`
- `removeFormat`

editor-engineer가 한 군데서 dispatch:
```js
formatBar.addEventListener('click', (e) => {
  const cmd = e.target.dataset.formatCmd;
  if (!cmd) return;
  // value가 있는 경우(font family, font size, color)는 e.target.value 사용
  // text-format.js의 해당 함수 호출
});
```

## 4. CSS 클래스 명세 (editor-engineer가 작성, 모든 모듈이 사용)

- `.htmledit-find-highlight` — 검색 매치 노란 형광. 직렬화 시 unwrap.
- `.htmledit-find-current` — 현재 포커스된 매치 (주황 외곽)
- `#mini-toolbar.visible` — 표시 상태
- `#format-bar` — 상단 서브툴바
- `.format-btn` — 서브툴바/미니툴바 공통 버튼 스타일
- `.format-btn[aria-pressed="true"]` — 활성 상태 표시
- `body.htmledit-alt-pressed` — Alt 키 눌린 상태 (drag.js와 editor.js가 사용)

## 5. 단축키 등록 위치 — 충돌 방지

| 키 | 등록처 | 비고 |
|----|--------|------|
| ⌘O / ⌘S / ⌘⇧S | 셸 + 메뉴 가속기 | 메뉴가 잡고 셸에서 별도 등록 X |
| ⌘Z / ⌘⇧Z | 셸 keydown + iframe keydown | 양쪽 등록 (이미) |
| ⌘⇧E | 셸 keydown | (옛 ⌘E → ⌘⇧E로 이동) |
| ⌘B/I/U/⇧X/=/⇧= | iframe keydown만 (text-format.js) | 셸엔 등록 X |
| ⌘L/E/R/J | iframe keydown만 | 정렬 |
| ⌘F/G/⇧G/⇧H | 셸 keydown만 (find-replace.js의 setup) | find 바가 셸에 있으므로 |
| ⌘]/⌘[/⌘\\ | iframe keydown만 | 들여쓰기·서식 지우기 |

iframe 키보드 핸들러는 capture phase에 등록해 사용자 HTML의 자체 핸들러보다 먼저 실행.

## 6. 메뉴 항목 추가 (electron-architect)

- **파일**: 백업으로 되돌리기... (새), Finder에서 보기 (새)
- **편집**: 찾기 ⌘F, 바꾸기 ⌘⇧H, 다음 찾기 ⌘G, 이전 찾기 ⌘⇧G
- **서식** (새): 굵게, 기울임, 밑줄, 취소선, 첨자/위첨자, 글자 색, 형광펜, 정렬, 글머리/번호, 들여쓰기, 서식 지우기, 폰트 (각 단축키 표기)
- **보기**: 편집 모드 켜기/끄기 ⌘⇧E (단축키 변경)

각 항목은 menu-action 채널로 렌더러에 통보 — 기존 패턴 그대로.

## 8. 백업 위치 명세 (fs-keeper)

```
~/Library/Application Support/Text Touch/backups/
└── {sha1(originalPath).slice(0,12)}/
    ├── original.txt       ← 원본 절대 경로 기록 (복원 시 검증)
    ├── 20260531-2240.html ← 백업 본
    ├── 20260531-2245.html
    └── 20260531-2310.html  (최근 5개 유지, 더 오래된 건 회전)
```

복원 시 `original.txt`와 사용자 지정 target이 일치하지 않으면 경고 모달.

## 9. 변경 금지 영역 (각 에이전트별)

- electron-architect는 `app/src/renderer/` 절대 건드리지 않음
- editor-engineer는 `app/src/{main,menu,ipc,fs-handlers,preload}.js` 절대 건드리지 않음
- fs-keeper는 `app/src/renderer/` + `main.js` 절대 건드리지 않음
- text-format-engineer는 `editor.js`, `drag.js`, `main.js`, `menu.js`, `ipc.js`, `fs-handlers.js`, `preload.js` 절대 건드리지 않음

마크업과 CSS는 editor-engineer 전담. JS 모듈은 text-format-engineer.

## 10. 충돌 시 우선순위

1. 이 contract를 따라간다. 본문이 모호하면 SendMessage로 team lead 호출.
2. 모듈 signature 변경 필요 시 contract 먼저 갱신 → 다른 에이전트 통보.
3. 새 의존성(npm install)이 필요하면 team lead가 직접 처리 (iconv-lite는 fs-keeper가 요청).
