# Text Touch v0.6 — API Contract (단일 진실 원천)

> 4가지 작업 병렬: 사용자 HTML base 주입(#1), 최근 파일 메뉴(#2), 이미지 alt 패널(#3), CRLF 전체 스캔(#8).

## 1. IPC 채널 — 신규

| 채널 | 방향 | 페이로드 | 응답 | 담당 |
|------|-----|---------|------|------|

기존 v0.5 채널은 그대로.

## 2. fs-handlers.loadFile 응답 변경 (fs-keeper)

기존 응답에 **`dir`** 필드 추가:
```ts
{
  path: string,
  dir: string,              // 신규: 원본 파일의 부모 디렉터리 절대 경로
  html: string,
  encoding: string,

  legacyEncodingWarn: boolean

```

`saveOriginal` / `saveAs` 페이로드 / 응답은 그대로.

## 3. CRLF 전체 스캔 (fs-keeper)

`detectLineEnding(text)`을 다음 규칙으로 강화:
- 전체 텍스트 스캔 (1MB 이상이면 16KB 샘플 3구간: 처음·중간·끝)
- CRLF / LF 카운트 후 다수파 선택
- 동률 시 CRLF 우선(Windows 호환성 보존)
- 코멘트로 "왜 전체 스캔인가" 명시

## 4. 최근 파일 메뉴 (electron-architect)

- `app.addRecentDocument(path)` — 파일 열 때마다 호출. macOS는 자동으로 Dock·메뉴 등록.
- 메뉴 **파일** 서브 끝부분에 신규 항목:
  - "최근 파일" 서브메뉴 — `submenu: [{ role: 'recentDocuments' }, { type: 'separator' }, { role: 'clearRecentDocuments' }]`
- 윈도우는 v0.6에선 OS 자동 등록만(`app.addRecentDocument`로 충분). 자체 관리 UI는 v0.7 후보.

editor.js는 파일 로드 성공 시 `window.htmledit.addRecent(path)`를 호출 (preload 노출 필요).

## 5. 이미지 alt 패널 (text-format-engineer)

신규 파일 `app/src/renderer/image-alt-panel.js`:

```ts
window.htmleditAltPanel = {
  init(shellEl: HTMLElement, getDoc: () => Document, callbacks: {
    onChange?: () => void  // markDirty 호출용
  }): void
  open(): void              // 패널 열기. open 시 이미지 목록 새로 스캔
  close(): void
  toggle(): void
  isOpen(): boolean
};
```

내부 동작:
- iframe.contentDocument의 `<img>` 모두 순회 (script/style 안의 건 제외)
- 각 항목: 작은 썸네일(64px) + 파일명 + **alt 입력** + **title 입력**
- 입력 시 즉시 iframe 안 `<img>`의 속성 갱신 + `callbacks.onChange()`
- 패널 열림 시 셸 stage가 우측으로 240px만큼 줄어듦 (CSS body.alt-panel-open 클래스)

마크업: `<aside id="alt-panel" hidden>` 컨테이너만 editor-engineer가 index.html에 추가. 내부는 모듈이 채움.

CSS 클래스(editor-engineer):
- `body.alt-panel-open #stage { right: 240px; }`
- `#alt-panel` 우측 사이드 fixed 패널 디자인 (라이트 톤)
- `.alt-item` 각 이미지 행

토글 진입로:
- 보기 메뉴 → "이미지 alt 편집" (action: `'toggleAltPanel'`)
- 또는 툴바 우측에 작은 버튼

## 6. base 주입 (editor.js의 mountDocument 안에서)

`mountDocument(htmlString)` 시그너처는 그대로. 다만 `state.dir`(loadFile 응답의 dir)을 사용해 srcdoc 직전 HTML 문자열에 base 주입:

```js
function injectBase(htmlString, baseDir) {
  if (!baseDir || /<base\b/i.test(htmlString)) return htmlString;
  const fileUrl = 'file://' + baseDir.split('/').map(encodeURIComponent).join('/').replace(/^/, '') + '/';
  const baseTag = `<base href="${fileUrl}" data-htmledit-injected="base">`;
  if (/<head\b[^>]*>/i.test(htmlString)) {
    return htmlString.replace(/<head\b[^>]*>/i, m => m + baseTag);
  }
  if (/<html\b[^>]*>/i.test(htmlString)) {
    return htmlString.replace(/<html\b[^>]*>/i, m => m + '<head>' + baseTag + '</head>');
  }
  return '<!DOCTYPE html><head>' + baseTag + '</head>' + htmlString;
}
```

`serializeForSave`에서는 `[data-htmledit-injected="base"]` 요소를 제거 후 직렬화.
